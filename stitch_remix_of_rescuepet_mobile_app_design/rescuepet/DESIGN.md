## Brand & Style
The brand personality of the design system is centered on compassion, reliability, and modern efficiency. It serves as a bridge between animals in need and loving families, requiring an emotional response that feels both urgent and comforting. 

The aesthetic follows a **Corporate / Modern** style, specifically adhering to the Material 3 (M3) philosophy. This ensures a familiar, highly accessible interface for Android users. The design emphasizes clarity through generous whitespace, expressive typography, and a tactile feel achieved through tonal elevations. The interface should feel "alive" and responsive, using soft transitions and a warm, inviting atmosphere to reduce the stress associated with the adoption process.

## Colors
The palette is rooted in natural, earthy tones to evoke a sense of care and stability.
- **Primary (Terracotta):** Used for key actions, brand moments, and active states. It represents the warmth of a home.
- **Secondary (Sage Green):** Used for secondary actions and environmental/health-related information. It provides a calming counterpoint to the primary color.
- **Tertiary (Ocean Blue):** Used for informational accents and administrative tasks to maintain a professional, trustworthy tone.
- **Surface & Neutrals:** The background remains a clean white (#FFFFFF). Surface variations (Surface 1 through 5) are generated by overlaying the primary color at very low opacities (1% to 12%) to create depth without relying solely on shadows.

## Typography
Inter is utilized across all levels to ensure maximum legibility and a clean, technical but approachable feel. 
- **Headlines:** Use `headline-lg` for screen titles. On mobile devices, use the `-mobile` variant to prevent excessive line wrapping.
- **Body:** `body-lg` is the standard for long-form descriptions (e.g., pet bios). `body-md` is used for supporting text.
- **Labels:** Use `label-lg` for buttons and navigation items. `label-sm` is reserved for small tags or captions.
- **Hierarchy:** Ensure a clear contrast between `title-md` and `body-md` by utilizing the medium (500) weight for titles.

## Layout & Spacing
The layout follows a fluid 12-column grid for tablets and a 4-column grid for mobile devices. 
- **Rhythm:** An 8dp linear spacing scale is used to maintain vertical rhythm.
- **Margins:** A standard 16dp outer margin is used for mobile to maximize content area while preventing edge-bleeding. 
- **Touch Targets:** All interactive elements must maintain a minimum touch target size of 48x48dp, even if the visual representation is smaller.
- **Alignment:** Content should be left-aligned for readability, with cards utilizing `spacing.md` (16px) for internal padding.

## Elevation & Depth
Depth in the design system is conveyed using **Tonal Layers** supplemented by light **Ambient Shadows**. 
- **Level 0 (Flat):** Used for the main background.
- **Level 1 (Tonal):** Surfaces used for cards or search bars. These use a subtle shadow (Blur: 3dp, Y: 1dp, Opacity: 8% Black).
- **Level 2 (Raised):** Used for elements that require interaction focus, such as FABs (Floating Action Buttons).
- **Backdrop Blurs:** Used sparingly for modal overlays to maintain context of the underlying screen.

## Shapes
The shape language is friendly and soft, aligning with the "Rounded" configuration.
- **Small Components:** Buttons and Input Fields use `rounded-md` (8px).
- **Medium Components:** Cards and Dialogs use `rounded-lg` (16px).
- **Large Components:** Bottom sheets and large featured banners use `rounded-xl` (24px) for the top corners.
- **Chips:** Always use a full pill shape for status indicators.

## Components
- **Buttons:** 
  - *Primary Filled:* High emphasis, uses Primary color with white text. 
  - *Outlined:* Medium emphasis, uses a 1px border of the Primary color with Primary-colored text. 
- **Input Fields:** Outlined style with an 8px corner radius. The border turns Primary color on focus and Error color for validation failures. Labels should float to the top border on focus.
- **Cards:** Use Level 1 elevation. Feature the animal's image at the top with a 16px internal padding for text below.
- **Chips:** 
  - *Status:* Use Success (Green), Warning (Amber), or Error (Red) containers with high-contrast text for "Available," "Pending," or "Urgent" statuses.
- **Navigation:** Use a Bottom Navigation bar with active states highlighted by a tonal pill shape behind the icon, consistent with M3 standards.
- **Pet Profile Highlights:** Use a horizontal list of circular "Quick Stats" (e.g., Age, Weight, Breed) for fast scanning.
